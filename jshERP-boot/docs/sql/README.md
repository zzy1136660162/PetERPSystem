# 数据库变更脚本说明

> ⚠️ **重要提示**：所有数据库变更必须通过Navicat手动执行，禁止通过代码自动修改数据库结构

## 目录结构

```
sql/
├── README.md                           # 本说明文件
├── V1.0__init.sql                      # 初始数据库（管伊佳原始结构）
├── V1.1__add_approval_flow.sql         # 审批流程功能
├── V1.2__add_customer_follow_up.sql    # 客户跟进功能
├── V1.3__add_exchange_goods.sql        # 换货功能
├── V1.4__add_sales_target.sql          # 销售目标功能
└── rollback/                           # 回滚脚本目录
    ├── R1.1__rollback_approval_flow.sql
    ├── R1.2__rollback_customer_follow_up.sql
    └── ...
```

## 命名规范

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 初始化脚本 | `V1.0__init.sql` | `V1.0__init.sql` |
| 功能脚本 | `V1.x__功能名称.sql` | `V1.1__add_approval_flow.sql` |
| 回滚脚本 | `R1.x__rollback_功能名称.sql` | `R1.1__rollback_approval_flow.sql` |

## 执行顺序

1. **V1.0__init.sql** - 首次部署时执行（管伊佳原始数据库结构）
2. **V1.1__add_approval_flow.sql** - 审批流程功能
3. **V1.2__add_customer_follow_up.sql** - 客户跟进功能
4. **V1.3__add_exchange_goods.sql** - 换货功能
5. **V1.4__add_sales_target.sql** - 销售目标功能

## 执行方式

### 方式一：Navicat 查询编辑器执行

1. 打开 Navicat
2. 连接到目标数据库
3. 点击 "查询" → "新建查询"
4. 复制SQL脚本内容粘贴到编辑器
5. 点击 "运行" 执行

### 方式二：Navicat 运行SQL文件

1. 打开 Navicat
2. 连接到目标数据库
3. 右键点击数据库 → "运行SQL文件"
4. 选择对应的 `.sql` 文件
5. 点击 "开始" 执行

## 执行后验证

每个脚本执行后，建议执行以下验证：

```sql
-- 查看新增表结构
DESCRIBE 新表名;

-- 查看新增字段
DESCRIBE 原表名;

-- 查看表数据（抽样）
SELECT * FROM 新表名 LIMIT 5;
```

## 注意事项

1. **禁止删除操作**：不允许删除表、字段、索引
2. **禁止修改字段名**：如需修改，建议新增字段并迁移数据
3. **保留回滚脚本**：每个变更脚本需配套回滚脚本
4. **记录执行日志**：记录执行时间、执行人、执行结果
5. **测试环境先行**：所有脚本必须在测试环境验证通过后才能上生产
